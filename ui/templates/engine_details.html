Decision Engine Details

How LeaseLens Works

LeaseLens combines location scoring, customer-fit analysis, and launch planning to turn a retail brief into a shortlist and an execution path.

Live System

BigQuery-backed retail intelligence, weighted evaluation, and planning output in one workflow.

Evaluation Flow

This page exists for reviewers who want to inspect the orchestration and scoring logic behind the recommendation output.

Stage 1

Brief Intake

The coordinator converts business type, budget, target customer, and area constraints into a structured evaluation request.

Stage 2

Location Intelligence

BigQuery tables score Bangalore micro-markets using demand, accessibility, market intensity, and positioning cues.

Stage 3

Customer Fit

The shortlist is interpreted through customer-fit and commercial risk signals so the recommendation reflects the business brief.

Stage 4

Launch Planning

The final output becomes an action plan with milestone, checklist, and validation steps rather than a static ranked list.

Data Sources

LeaseLens relies on structured tables instead of prompt-only outputs.

Area Scores

area_live_scores_serving provides the cleaned serving-layer view of Bangalore market candidates.

Business Profiles

business_profiles provides the display label, fit text, and positioning used to contextualize each recommendation.

Rendered Output

The UI sends the brief to the backend coordinator API, then renders the returned shortlist, decision snapshot, execution plan, and Google Maps handoff.

Challenge Mapping

This implementation is deliberately structured so the technical requirements are visible in the product and codebase.

Primary Agent + Sub-Agents

The live ADK backend uses a coordinator agent and a sequential workflow of specialist agents for location analysis, customer fit, and launch planning.

Structured Database

BigQuery is used for both retrieval of retail intelligence and persistence of saved expansion sessions.

Multiple MCP Tools

The backend is wired to BigQuery MCP and Google Maps MCP so reasoning and validation can use multiple external tools.

Stored Workflow State

Each expansion run is saved as structured data so the system does not only retrieve intelligence, it also persists decision context for follow-up work.
